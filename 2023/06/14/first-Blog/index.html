一直希望记录学习的过程，生活的点滴，尝试过word文档，微博，杂乱，易丢，这次尝试回到bolg

1.准备工作

安装node.js下载地址-http://nodejs.cn

安装Git下载地址-Git

2.让git与你的GitHub建立连接，或者说是在 Git 中设置用户名

执行完毕后将在你的用户目录下生成.gitconfig文件，记录你在GitHub的用户名及邮箱

3.通过git生成ssh文件，并创建公钥,用于服务器

检查ssh keys是否存在，/ 代表用户根目录，在windows的C：\用户\用户名

用于生成rsa协议的ssh keys，过程中会有三次停顿，可以连续按回车
密钥用于鉴权，请谨慎保管。公钥文件以 .pub 扩展名结尾，可以公开给其他人，而没有 .pub 扩展名的私钥文件不要泄露给任何人！
也可以把你其他机器上的.ssh文件夹复制到 你的windows用户目录下，利用网盘，云同步等，实现跨机器Hexo博客发布

4.在Github创建 repository

登陆Github,点击Create repository，选中Public，名字填：用户名.github.io 。点击create repository

5.SSH添加到GitHub

在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key 把你刚才生成的id_rsa.pub里面的信息复制进去。

输入命令：ssh -T git@github.com 并且输入yes之后，行末尾显示你的用户名，说明添加成功。

安装cnpm,镜像选用https://registry.npm.taobao.org

用cnpm代替npm，安装 hexo

如何插入图片

绝对引用：当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们
source/images/image.jpg

相对引用：图片除了可以放在统一的images文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置_config.yml来生成。

将_config.yml文件中的配置项post_asset_folder设为true后，执行命令$ hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。

上述是markdown的引用方式，图片只能在文章中显示，但无法在首页中正常显示。

如果希望图片在文章和首页中同时显示，可以使用标签插件语法。
